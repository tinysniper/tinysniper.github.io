问题

npm安装electron-prebuil超级慢。

系统环境

OS X Yosemite。

缘起

看到electron，感觉挺新奇的，想尝试一下，于是拿electronic-wechat做下测试。
npm这些工具都安装正常，按照操作步骤执行npm install，结果在下载electron-prebuilt时挂着一直下载不了。

electron安装问题

之前已经安装了Electron.app，但编译时使用的是命令行的electron工具，所以在npm install时又对electron-prebuilt进行下载。
然而下载速度一直是0，刚开始以为是网络问题，后来查询发现下载不了是墙内的普遍现象。

尝试的解决办法

既然是因为安装包源访问的限制，我们可以尝试用墙内可访问的地址。查询到有如下两个地址：

https://registry.npm.taobao.org
http://cnpmjs.org

使用新安装源的方法：

  1. 修改.npmrc文件
    修改~/.npmrc，把地址替换为上面的任意一个：

    registry=https://registry.npm.taobao.org

  2. npm命令

理论上，上述两种方法对安装源的设定都是可以的，执行npm install应该成功。但现实很残酷，还是无法安装electron-prebuilt！
折腾了N久，也尝试重启了电脑，还是无法安装。(＞﹏＜)

最终解决办法

也算功夫不负有心人。今天又在查找答案，发现如果使用国内安装源，应该使用cnpm，而不是npm。于是安装cnpm：

然后再执行cnpm install，速度刷刷的，安装成功。

nrm

nrm是npm的开源包，就是为了解决npm安装源的问题，cnpm install -g nrm安装后，可以很方便的查看国内可用的安装源，也可以使用该命令切换不同的安装源。
